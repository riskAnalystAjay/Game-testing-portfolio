 Contributing Guidelines

Thank you for considering contributing to this Game Testing Portfolio.

This project documents manual QA testing practice, including test cases, bug reports, checklists, and summaries.

 How to Contribute
- Provide improvements to test case structure
- Suggest additional gameplay scenarios to test
- Improve formatting or documentation
- Submit new checklists or bug examples

 Process
1. Fork the repository.
2. Create a feature branch.
3. Commit your changes with a clear message.
4. Submit a pull request for review.

Notes
This is a learning-focused project. All contributions should follow standard QA practices with clear, reproducible steps.

Thank you for your support!
